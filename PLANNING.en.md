# Senior Planning — Kickoff & Delivery SOP

**Languages:** [English](PLANNING.en.md) · [中文](PLANNING.md)

> Read after scaffold. Agents follow `rules/senior-planning.mdc`.

## Core ideas

1. **Dual metrics:** `code_complete` ≠ `go_to_market` — track separately in `SUMMARY.md` / `ROADMAP.md`.
2. **Scope lock:** No v1.1 / add-ons until paid client or `artifacts/decision_*.md`.
3. **Ship order:** deploy → public URL → real dependency smoke test → outreach → inquiry.
4. **Tech debt:** stubs, fixture-only tests, local URLs → `TECH_DEBT.md`.
5. **Time honesty:** human hours vs agent compute vs equivalent dev hours — don’t conflate.
6. **When code ≥70% and GTM <40%:** stop features; run `docs/go-to-market-checklist.md`.
